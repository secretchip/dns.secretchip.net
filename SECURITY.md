# Security Policy

## Reporting a Vulnerability

Do not disclose security vulnerabilities in public GitHub issues.

Use one of the following channels:

1. GitHub Private Vulnerability Reporting  
   Use the "Report a vulnerability" option in the repository Security tab.

2. Email  
   security@secretchip.net

## What to Include

- High-level description of the issue
- Affected component or endpoint
- Impact assessment
- Minimal proof-of-concept explanation without weaponization
- Disclosure timeline constraints if applicable

## What NOT to Include in Public

- Exploit steps or payloads
- Credentials, tokens, API keys
- Customer data
- Full logs or memory dumps

## Our Commitment

- We will acknowledge receipt.
- We will coordinate remediation and disclosure.
- Good-faith research conducted under this policy is authorized.

## Safe Harbor

If you follow this policy, we will not pursue legal action for your security research.
